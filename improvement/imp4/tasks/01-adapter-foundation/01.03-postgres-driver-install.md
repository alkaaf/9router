## Task Description

Add the `pg` (node-postgres) npm package as a project dependency, then verify the installation by running a minimal smoke test. This is a prerequisite for Domain 1.01 (the adapter implementation).

## Context

The 9Router project currently has zero PostgreSQL dependencies. Adding `pg` is a small but critical change. `pg` is the de-facto PostgreSQL client for Node.js, used by Prisma, Knex, and most production Node deployments. It supports connection pooling, prepared statements, and parameter binding out of the box.

## Implementation Steps

### Step 1: Install pg
```bash
cd /Users/alkaaf/project/9router
npm install pg
grep '"pg"' package.json
```

### Step 2: Verify installation
```bash
node -e "console.log(require('pg/package.json').version)"
```

### Step 3: Smoke test (if Docker available)
```bash
docker run -d --name pg-smoke -e POSTGRES_PASSWORD=test -p 5432:5432 postgres:16-alpine
sleep 3
node -e "
  const { Client } = require('pg');
  const c = new Client({ connectionString: 'postgres://postgres:test@localhost:5432/postgres' });
  c.connect().then(() => c.query('SELECT 1 as n')).then(r => { console.log(r.rows); c.end(); });
"
docker rm -f pg-smoke
```

### Step 4: Verify lockfile
```bash
grep -A 2 '"pg":' package-lock.json | head -5
```

## Files to Modified
