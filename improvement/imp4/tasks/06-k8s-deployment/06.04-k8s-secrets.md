## Task Description

Create K8s Secret and ConfigMap manifests for 9Router deployment. The Secret contains `POSTGRES_URL` (and PG credentials if using the self-hosted StatefulSet). The ConfigMap contains non-sensitive config.

## Context

Secrets in K8s are base64-encoded but NOT encrypted at rest by default. For production, use Sealed Secrets, External Secrets Operator, or cloud-provider secret managers. This task creates the basic manifest; production hardening is documented in the deployment guide.

## Implementation Steps

### Step 1: Create postgres-secret.yaml (PG credentials)
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: postgres-secret
type: Opaque
stringData:
  POSTGRES_DB: "9router"
  POSTGRES_USER: "9router"
  POSTGRES_PASSWORD: "change-me-in-production"  # use sealed-secrets or external-secrets in prod
```

### Step 2: Create 9router-secret.yaml (POSTGRES_URL)
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: 9router-secret
type: Opaque
stringData:
  # Replace with the actual connection string
  # In production, generate from sealed-secrets or external-secrets
  POSTGRES_URL: "postgres://9router:change-me-in-production@postgres:5432/9router"
```

### Step 3: Create 9router-configmap.yaml (non-sensitive config)
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: 9router-config
data:
  POSTGRES_POOL_SIZE: "10"
  LOG_LEVEL: "info"
  PORT: "20128"
```

### Step 4: Apply and verify
```bash
kubectl apply -f deploy/k8s/postgres-secret.yaml
kubectl apply -f deploy/k8s/9router-secret.yaml
kubectl apply -f deploy/k8s/9router-configmap.yaml
# Verify Secret is mounted
kubectl exec -it <9router-pod> -- env | grep POSTGRES
# Should show POSTGRES_URL (with password masked in describe)
```

## Files to Create

| File | Content |
|------|---------|
| deploy/k8s/postgres-secret.yaml | PG credentials |
| deploy/k8s/9router-secret.yaml | POSTGRES_URL |

## Success Criteria

- [ ] Secrets are created (not visible in `kubectl get secret -o yaml` by default)
- [ ] ConfigMap values appear as env vars in 9Router pods
- [ ] `POSTGRES_URL` is mounted correctly
- [ ] `kubectl describe pod` masks the Secret values

## Testing Checklist

- [ ] Apply all 3 manifests
- [ ] 9Router pods start successfully (env vars are present)
- [ ] Connection from 9Router to PostgreSQL works
- [ ] `kubectl describe pod` shows the env vars but values are masked

## Notes

- For production: use Sealed Secrets (bitnami-labs/sealed-secrets) or External Secrets Operator
- The `change-me-in-production` placeholder MUST be replaced before applying
- `stringData` accepts plain text (base64 encoding is automatic)
- Document the secret rotation procedure in the deployment guide
- Consider generating the password with `openssl rand -base64 32`
