## Task Description

Create K8s Service manifests: a headless Service for the PostgreSQL StatefulSet (for stable DNS), a ClusterIP Service for 9Router (in-cluster access), and an optional LoadBalancer variant for external access.

## Context

A headless Service (`clusterIP: None`) is required by StatefulSets for stable DNS entries (`postgres-0.postgres`). A regular ClusterIP Service is sufficient for in-cluster access to 9Router; a LoadBalancer variant provides external access.

## Implementation Steps

### Step 1: Create postgres-service.yaml (headless)
```yaml
apiVersion: v1
kind: Service
metadata:
  name: postgres
  labels: { app: postgres }
spec:
  clusterIP: None
  selector: { app: postgres }
  ports:
    - name: postgres
      port: 5432
      targetPort: 5432
```

### Step 2: Create 9router-service.yaml (ClusterIP)
```yaml
apiVersion: v1
kind: Service
metadata:
  name: 9router
  labels: { app: 9router }
spec:
  type: ClusterIP
  selector: { app: 9router }
  ports:
    - name: http
      port: 20128
      targetPort: 20128
```

### Step 3: Create 9router-service-lb.yaml (LoadBalancer variant)
```yaml
apiVersion: v1
kind: Service
metadata:
  name: 9router-external
  labels: { app: 9router }
spec:
  type: LoadBalancer
  selector: { app: 9router }
  ports:
    - name: http
      port: 80
      targetPort: 20128
```

### Step 4: Verify DNS resolution
```bash
kubectl apply -f deploy/k8s/postgres-service.yaml
kubectl apply -f deploy/k8s/9router-service.yaml
# DNS check from inside a pod:
kubectl run -it --rm debug --image=busybox --restart=Never -- nslookup postgres-0.postgres
# Should return the StatefulSet pod's IP
```

### Step 5: Test in-cluster access
```bash
kubectl run -it --rm curl-test --image=curlimages/curl --restart=Never -- \
  curl http://9router:20128/api/health
# Should return 200
```

## Success Criteria

- [ ] Headless Service creates stable DNS for PG pods
- [ ] 9Router ClusterIP Service is reachable from other pods
- [ ] LoadBalancer variant works (when applied to a cluster with LB support)
- [ ] DNS resolution: `nslookup postgres-0.postgres` returns the pod IP

## Testing Checklist

- [ ] Apply all 3 Services
- [ ] DNS resolution works from another pod
- [ ] 9Router ClusterIP Service responds on port 20128
- [ ] LoadBalancer service (if supported) gets an external IP

## Notes

- Headless Service is REQUIRED for StatefulSet pod DNS to work
- The 9Router pods reference `postgres` Service by name in the POSTGRES_URL secret
- LoadBalancer is optional -- for production, prefer Ingress with TLS termination
- The port mapping 80 -> 20128 in the LB is a common convention; adjust as needed
