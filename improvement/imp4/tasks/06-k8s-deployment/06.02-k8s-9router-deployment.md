## Task Description

Create `deploy/k8s/9router-deployment.yaml` for running 9Router with 2-3 replicas. Includes resource requests/limits, env vars from Secret + ConfigMap, and liveness/readiness probes against `/api/health`.

## Context

The 9Router Deployment uses the 9Router container image. With PostgreSQL, it can run multiple replicas safely (the JSON-blob race condition is fixed by Domain 5). The `POSTGRES_URL` is provided via a Secret (created in 06.04). The `terminationGracePeriodSeconds: 30` allows the in-process write queue to flush on shutdown.

## Implementation Steps

### Step 1: Create the Deployment manifest
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: 9router
  labels: { app: 9router }
spec:
  replicas: 2
  selector:
    matchLabels: { app: 9router }
  template:
    metadata:
      labels: { app: 9router }
    spec:
      terminationGracePeriodSeconds: 30
      containers:
        - name: 9router
          image: 9router:latest  # update with actual image
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 20128
          env:
            - name: POSTGRES_URL
              valueFrom:
                secretKeyRef: { name: 9router-secret, key: POSTGRES_URL }
            - name: POSTGRES_POOL_SIZE
              valueFrom:
                configMapKeyRef: { name: 9router-config, key: POSTGRES_POOL_SIZE }
            - name: NODE_ENV
              value: production
            - name: PORT
              value: "20128"
            - name: LOG_LEVEL
              valueFrom:
                configMapKeyRef: { name: 9router-config, key: LOG_LEVEL }
          resources:
            requests: { cpu: 200m, memory: 512Mi }
            limits:   { cpu: "1", memory: 2Gi }
          livenessProbe:
            httpGet: { path: /api/health, port: 20128 }
            initialDelaySeconds: 30
            periodSeconds: 10
            timeoutSeconds: 3
            failureThreshold: 3
          readinessProbe:
            httpGet: { path: /api/health, port: 20128 }
            initialDelaySeconds: 5
            periodSeconds: 5
            timeoutSeconds: 3
            failureThreshold: 3
          startupProbe:
            httpGet: { path: /api/health, port: 20128 }
            initialDelaySeconds: 10
            periodSeconds: 5
            failureThreshold: 30
```

### Step 2: Verify the health endpoint exists
```bash
# Confirm /api/health returns 200 in the codebase
grep -rn "/api/health" src/app/api/
```

### Step 3: Apply and verify
```bash
kubectl apply -f deploy/k8s/9router-deployment.yaml
kubectl wait --for=condition=ready pod -l app=9router --timeout=120s
kubectl port-forward svc/9router 20128:20128
# Open http://localhost:20128 -- dashboard should load
```

## Notes

- 2 replicas is the minimum for HA; 3 is recommended for production
- The startup probe gives the app up to 150s to start (30 failures x 5s) -- critical for first init
- `terminationGracePeriodSeconds: 30` must match or exceed the app's flush timeout
- The image tag `9router:latest` should be replaced with a real image in production
- Consider `podDisruptionBudget` to prevent all pods going down during node drains
