## Task Description

Create `src/lib/db/schema.postgres.js` containing the 12 PostgreSQL-native table definitions. Each table mirrors its SQLite counterpart but uses PostgreSQL types: JSONB for JSON, TIMESTAMPTZ for timestamps, BOOLEAN for booleans, BIGSERIAL for auto-increment IDs, and NUMERIC(12,6) for monetary cost.

## Context

The SQLite schema (`src/lib/db/schema.js`) is loosely typed. PostgreSQL enforces stricter types, and using native types unlocks performance and correctness gains. The PostgreSQL schema is a parallel definition -- the SQLite schema is never modified. The driver selects which schema to apply at runtime.

The 12 tables: _meta, settings, providerConnections, providerNodes, proxyPools, apiKeys, combos, kv, usageHistory, usageDaily, requestDetails. The 5 normalized rollup tables are added in Domain 5.

## Implementation Steps

### Step 1: Create the file with header
```js
// src/lib/db/schema.postgres.js
export const SCHEMA_VERSION = "1.0.0";
export const PRAGMA_SQL_POSTGRES = `SET synchronous_commit = 'on'; SET timezone = 'UTC';`;
export const TABLES_POSTGRES = [ /* ... */ ];
```

### Step 2: Define each table
Each table gets a CREATE TABLE IF NOT EXISTS statement:
- `_meta`: key TEXT PK, value TEXT NOT NULL
- `settings`: id INTEGER PK CHECK(id=1), data JSONB NOT NULL
- `providerConnections`: id TEXT PK, provider TEXT, authType TEXT, name TEXT, email TEXT, priority INTEGER, isActive BOOLEAN DEFAULT TRUE, data JSONB NOT NULL, createdAt TIMESTAMPTZ DEFAULT NOW(), updatedAt TIMESTAMPTZ DEFAULT NOW()
- `providerNodes`: id TEXT PK, type TEXT, name TEXT, data JSONB NOT NULL, createdAt TIMESTAMPTZ DEFAULT NOW(), updatedAt TIMESTAMPTZ DEFAULT NOW()
- `proxyPools`: id TEXT PK, isActive BOOLEAN DEFAULT TRUE, testStatus TEXT, data JSONB NOT NULL, createdAt TIMESTAMPTZ DEFAULT NOW(), updatedAt TIMESTAMPTZ DEFAULT NOW()
- `apiKeys`: id TEXT PK, key TEXT UNIQUE NOT NULL, name TEXT, machineId TEXT, isActive BOOLEAN DEFAULT TRUE, createdAt TIMESTAMPTZ DEFAULT NOW()
- `combos`: id TEXT PK, name TEXT UNIQUE NOT NULL, kind TEXT, models JSONB NOT NULL, createdAt TIMESTAMPTZ DEFAULT NOW(), updatedAt TIMESTAMPTZ DEFAULT NOW()
- `kv`: scope TEXT, key TEXT, value TEXT, PRIMARY KEY(scope, key)
- `usageHistory`: id BIGSERIAL PK, timestamp TIMESTAMPTZ, provider TEXT, model TEXT, connectionId TEXT, apiKey TEXT, endpoint TEXT, promptTokens BIGINT DEFAULT 0, completionTokens BIGINT DEFAULT 0, cost NUMERIC(12,6) DEFAULT 0, status TEXT, tokens JSONB, meta JSONB
- `usageDaily`: dateKey DATE PK, data JSONB NOT NULL
- `requestDetails`: id TEXT PK, timestamp TIMESTAMPTZ, provider TEXT, model TEXT, connectionId TEXT, status TEXT, data JSONB NOT NULL

### Step 3: Verify by applying against test DB
```bash
# Use docker-compose.test.yml (Domain 8) to start PG
node -e "
  import('./src/lib/db/schema.postgres.js').then(m => {
    console.log('Tables:', m.TABLES_POSTGRES.length);
    console.log('Version:', m.SCHEMA_VERSION);
  });
"
```

## Notes

- The 5 normalized rollup tables (usage_daily_by_*) are added in Domain 5
- CHECK (id = 1) on settings is critical -- enforces the single-row pattern
- Use IF NOT EXISTS everywhere so re-running init is safe
- TIMESTAMPTZ stores all timestamps in UTC
