## Task Description

Ensure the PostgreSQL adapter's init() seeds the default _meta rows (SCHEMA_VERSION, totalRequestsLifetime) and the default settings row -- mirroring SQLite init behavior.

## Context

The SQLite init path checks for and inserts default rows on first run:
- _meta: { key: 'SCHEMA_VERSION', value: '<version>' }
- _meta: { key: 'totalRequestsLifetime', value: '0' }
- settings: { id: 1, data: { ...DEFAULT_SETTINGS } }

The PostgreSQL init must do the same. Use INSERT ... ON CONFLICT DO NOTHING for idempotency.

## Implementation Steps

### Step 1: Identify SQLite seeding logic
```bash
grep -n "INSERT.*_meta\|INSERT.*settings\|DEFAULT_SETTINGS" src/lib/db/index.js
```

### Step 2: Add seed function
```js
const DEFAULT_META_ROWS = [
  { key: 'SCHEMA_VERSION', value: SCHEMA_VERSION },
  { key: 'totalRequestsLifetime', value: '0' },
];

async function seedPostgresDefaults(adapter) {
  for (const row of DEFAULT_META_ROWS) {
    await adapter.run(
      "INSERT INTO _meta (key, value) VALUES ($1, $2) ON CONFLICT (key) DO NOTHING",
      [row.key, row.value]
    );
  }
  await adapter.run(
    "INSERT INTO settings (id, data) VALUES ($1, $2) ON CONFLICT (id) DO NOTHING",
    [1, DEFAULT_SETTINGS]
  );
}
```

### Step 3: Wire into initSchema
```js
if (adapter.driver === "postgres") {
  await adapter.exec(PRAGMA_SQL_POSTGRES);
  for (const table of TABLES_POSTGRES) await adapter.exec(table);
  for (const index of INDEXES_POSTGRES) await adapter.exec(index);
  await seedPostgresDefaults(adapter);
}
```

### Step 4: Verify idempotency
```bash
# Run init twice -- should not duplicate rows
SELECT COUNT(*) FROM _meta; -- should be 2
SELECT COUNT(*) FROM settings; -- should be 1
```

## Notes

- totalRequestsLifetime must be stored as TEXT string -- usageRepo atomic counter casts it to bigint in SQL
- ON CONFLICT DO NOTHING is simpler than DO UPDATE for re-init safety
- The adapter may auto-serialize DEFAULT_SETTINGS objects to JSONB -- verify behavior
