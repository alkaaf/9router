## Task Description

Replace the per-row `insertStmt.run(...)` loop in `_flushWriteQueue` with a single multi-row INSERT into `usageHistory`. Build the VALUES clause dynamically from the entries array. This reduces 50+ round-trips per batch to 1.

## Context

Currently, `_flushWriteQueue` iterates over pending entries and calls `insertStmt.run(...)` for each. PostgreSQL can accept a single INSERT with multiple VALUES tuples: `INSERT INTO t (a, b) VALUES ($1, $2), ($3, $4), ...`. This is dramatically faster for batch flushes.

## Implementation Steps

### Step 1: Read current _flushWriteQueue
```bash
# In src/lib/db/repos/usageRepo.js
grep -n "insertStmt\|_flushWriteQueue\|INSERT INTO usageHistory" src/lib/db/repos/usageRepo.js
```

### Step 2: Write a multi-row INSERT builder
```js
function buildMultiRowInsert(table, columns, rows) {
  if (rows.length === 0) return null;
  const placeholders = rows.map((_, i) => 
    `(${columns.map((_, j) => `$${i * columns.length + j + 1}`).join(", ")})`
  ).join(", ");
  const sql = `INSERT INTO ${table} (${columns.join(", ")}) VALUES ${placeholders}`;
  const params = rows.flatMap(row => columns.map(c => row[c]));
  return { sql, params };
}
```

### Step 3: Apply in _flushWriteQueue
```js
// For usageHistory:
const { sql, params } = buildMultiRowInsert(
  "usageHistory",
  ["timestamp", "provider", "model", "connectionId", "apiKey", "endpoint",
   "promptTokens", "completionTokens", "cost", "status", "tokens", "meta"],
  entries
);
if (sql) await db.exec(sql, params);
```

### Step 4: Apply for usageDaily UPSERT
```sql
INSERT INTO usageDaily (dateKey, data) VALUES ($1, $2), ($3, $4), ...
ON CONFLICT (dateKey) DO UPDATE SET data = EXCLUDED.data
```
