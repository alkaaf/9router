## Task Description

Replace the monolithic `usageDaily.data` JSON blob with 5 normalized rollup tables that enable concurrent incremental updates. Each table has a composite PK on (date_key, dimension) and uses `INSERT ... ON CONFLICT DO UPDATE` for atomic aggregation.

## Context

The current pattern `INSERT OR REPLACE INTO usageDaily ... SET data = excluded.data` does a full JSON clobber per flush. With multiple pods flushing concurrently, this loses updates (last writer wins). Normalized tables fix this: each row is updated independently with `requests = table.requests + EXCLUDED.requests`.

The 5 tables:
- `usage_daily_by_provider` (PK: date_key, provider)
- `usage_daily_by_model` (PK: date_key, model, provider)
- `usage_daily_by_account` (PK: date_key, connectionId)
- `usage_daily_by_api_key` (PK: date_key, apiKey)
- `usage_daily_by_endpoint` (PK: date_key, endpoint, model, provider)

Each row: requests BIGINT, promptTokens BIGINT, completionTokens BIGINT, cost NUMERIC(12,6), lastUsed TIMESTAMPTZ.

### Step 1: Add the 5 tables to schema.postgres.js
```sql
CREATE TABLE IF NOT EXISTS usage_daily_by_provider (
  date_key DATE NOT NULL,
  provider TEXT NOT NULL,
  requests BIGINT DEFAULT 0,
  promptTokens BIGINT DEFAULT 0,
  completionTokens BIGINT DEFAULT 0,
  cost NUMERIC(12,6) DEFAULT 0,
  lastUsed TIMESTAMPTZ,
  PRIMARY KEY (date_key, provider)
);

-- usage_daily_by_endpoint (PK: date_key, endpoint, model, provider)
```

### Step 2: Update aggregateEntryToDay() helper
```js
// Instead of building a nested JSON object, generate UPSERTs for each rollup table
function aggregateEntryToDay(entry) {
  const dateKey = entry.timestamp.toISOString().slice(0, 10);
  return {
    provider: { dateKey, dim: entry.provider, tokens: entry.promptTokens + entry.completionTokens, cost: entry.cost },
    model: { dateKey, dim1: entry.model, dim2: entry.provider, ... },
    // ... etc for the 5 tables
  };
}
```

### Step 3: Generate UPSERTs
```sql
INSERT INTO usage_daily_by_provider (date_key, provider, requests, promptTokens, completionTokens, cost, lastUsed)
VALUES ($1, $2, $3, $4, $5, $6, $7)
ON CONFLICT (date_key, provider) DO UPDATE SET
  requests = usage_daily_by_provider.requests + EXCLUDED.requests,
  promptTokens = usage_daily_by_provider.promptTokens + EXCLUDED.promptTokens,
  completionTokens = usage_daily_by_provider.completionTokens + EXCLUDED.completionTokens,
  cost = usage_daily_by_provider.cost + EXCLUDED.cost,
  lastUsed = GREATEST(usage_daily_by_provider.lastUsed, EXCLUDED.lastUsed);
```
