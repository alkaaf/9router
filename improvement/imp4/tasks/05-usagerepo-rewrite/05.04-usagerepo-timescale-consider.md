## Task Description

Replace the JS-side JSON aggregation in `getUsageStats` and `getChartData` with SQL aggregates over the normalized rollup tables. Output shape must be bit-identical to the SQLite path so the dashboard does not need to change.

## Context

Currently, `getUsageStats` loads N days of JSON blobs from `usageDaily`, parses them, and folds them into in-memory maps. With 5 rollup tables, this becomes 5 SQL queries (one per dimension) with `SUM()` aggregation -- much faster and PG-friendly.

`getChartData` for today/24h can use `date_trunc('hour', timestamp)` to bucket in SQL.

## Implementation Steps

### Step 1: Read current getUsageStats
```bash
grep -n "getUsageStats\|loadDaysInRange" src/lib/db/repos/usageRepo.js
```

### Step 2: Replace with SQL aggregation
```sql
-- Daily summary per provider:
SELECT provider, SUM(requests) as totalRequests, SUM(promptTokens) as totalPromptTokens,
       SUM(completionTokens) as totalCompletionTokens, SUM(cost) as totalCost
FROM usage_daily_by_provider
WHERE date_key >= $1
GROUP BY provider;
-- Repeat for the 4 other rollup tables
```

### Step 3: Merge results in JS
- Collect 5 query results
- Build the same `{ totalRequests, totalPromptTokens, ... byProvider, byModel, ... }` shape
- Bit-identical to the SQLite path

### Step 4: Optimize lastUsed overlay
```sql
-- Replace full scan with:
SELECT DISTINCT ON (apiKey) apiKey, timestamp, provider, model, connectionId, endpoint
FROM usageHistory
WHERE timestamp >= $1
ORDER BY apiKey, timestamp DESC;
```

### Step 5: Rewrite getChartData
```sql
-- For today/24h:
SELECT date_trunc('hour', timestamp) as hour,
       SUM(promptTokens) as promptTokens,
       SUM(completionTokens) as completionTokens,
       SUM(cost) as cost
FROM usageHistory
WHERE timestamp >= $1
GROUP BY hour
ORDER BY hour;
-- For 7d/30d/60d: same query but against usage_daily_by_provider (daily buckets)
```
