## Task Description

Write user-facing documentation: a PostgreSQL deployment guide (K8s self-hosted + managed PG), an environment variable reference, and a README update mentioning PostgreSQL as a supported backend.

## Context

Operators need clear, step-by-step instructions to deploy 9Router with PostgreSQL. The guide should cover both self-hosted (StatefulSet) and managed (RDS/Cloud SQL) PostgreSQL, since managed is the production recommendation.

## Implementation Steps

### Step 1: Create docs/postgresql-deployment.md
- **Prerequisites**: K8s cluster (or Docker Compose for dev), kubectl
- **Self-hosted PostgreSQL**:
  1. Apply postgres-pvc.yaml
  2. Apply postgres-secret.yaml (with your own password)
  3. Apply postgres-statefulset.yaml
  4. Apply postgres-service.yaml
  5. Wait for pod ready
- **Managed PostgreSQL** (recommended for production):
  1. Provision RDS/Cloud SQL/Aiven instance
  2. Note the connection string
  3. Skip the StatefulSet manifests
- **9Router deployment**:
  1. Apply 9router-secret.yaml (with POSTGRES_URL)
  2. Apply 9router-configmap.yaml
  3. Apply 9router-deployment.yaml
  4. Apply 9router-service.yaml
  5. Verify pods ready
- **Verification**:
  - `kubectl port-forward svc/9router 20128:20128`
  - Open http://localhost:20128
  - Create a test API key
  - Make a test API call
- **Scaling**: Update replicas in 9router-deployment.yaml

### Step 2: Create docs/postgresql-env-vars.md
- `POSTGRES_URL` (required for PG mode) - connection string
- `POSTGRES_POOL_SIZE` (default 20) - connection pool size
- `POSTGRES_SQL` (default false) - enable SSL
- Document SQLite fallback: when POSTGRES_URL is unset, the app uses SQLite automatically
- Example connection strings for:
  - Local Docker: `postgres://test:test@localhost:5432/9router`
  - AWS RDS: `postgres://user:pass@mydb.cluster-xxx.us-east-1.rds.amazonaws.com:5432/9router`
  - K8s in-cluster: `postgres://9router:pass@postgres:5432/9router`

### Step 3: Update README.md
Add a "PostgreSQL / Kubernetes" section:
- Brief description
- Link to deployment guide
- Minimum requirements (PostgreSQL 14+)
- Text-based architecture diagram
