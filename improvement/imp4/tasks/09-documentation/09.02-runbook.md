## Task Description

Write the SQLite -> PostgreSQL migration runbook, a troubleshooting guide for common issues, and a CHANGELOG entry describing the new PostgreSQL support.

## Context

Operators need:
1. A step-by-step runbook for migrating existing 9Router SQLite data to PostgreSQL
2. A troubleshooting guide for common issues
3. A clear changelog entry explaining the new capability

## Implementation Steps

### Step 1: Create docs/postgresql-migration-runbook.md
Step-by-step procedure:
1. **Prerequisites**: PostgreSQL instance running, network access from 9Router host
2. **Back up SQLite database**: `cp data.db data.db.bak`
3. **Install pg client libraries** if needed
4. **Run migration**: `node scripts/migrate-sqlite-to-postgres.js --sqlite ./data.db --postgres <dsn>`
5. **Verify**: check row counts, spot-check usage data
6. **Configure POSTGRES_URL** in 9Router env
7. **Start 9Router with POSTGRES_URL set**
8. **Monitor for 24h**, verify all features work
9. **Rollback procedure**: unset POSTGRES_URL -> auto-fallback to SQLite
10. **If rollback is needed**: run `migrate-postgres-to-sqlite.js` to copy data back

### Step 2: Create docs/postgresql-troubleshooting.md
Common issues with symptom -> cause -> fix:
- "9Router won't start with POSTGRES_URL set" -> check connection string format, network connectivity, PG running
- "Connection pool exhausted" -> increase POSTGRES_POOL_SIZE, check for connection leaks
- "Data looks wrong after migration" -> verify migration completed, check row counts, re-run migration
- "Performance is slower than SQLite" -> check connection pool, network latency, consider PgBouncer
- "Dashboard shows no usage data" -> verify usageHistory rows in PG, check adapter selection log
- "Multi-replica deployment shows inconsistent data" -> verify rollup tables are populated correctly
- "Migration script hangs" -> check usageHistory size, increase BATCH size, monitor memory

### Step 3: Update CHANGELOG.md
```markdown
## [X.Y.Z] -- 2026-06-03

### Added
- PostgreSQL adapter support -- set POSTGRES_URL to enable
- Kubernetes deployment manifests for multi-replica 9Router
- Data migration script: SQLite -> PostgreSQL
- Integration test suite for PostgreSQL

### Changed
- Usage aggregation now uses normalized rollup tables (PostgreSQL mode)
- Settings and API key lookups support PostgreSQL JSONB columns

### Migration
- Set POSTGRES_URL to migrate from SQLite to PostgreSQL
- Unset POSTGRES_URL to revert to SQLite (fallback is automatic)
- See docs/postgresql-migration-runbook.md for step-by-step procedure
```
