## Task Description

Guard the existing PRAGMA_SQL execution in the adapter init path so that SQLite-specific PRAGMAs (WAL, busy_timeout, cache_size, etc.) are NOT executed when the PostgreSQL adapter is active.

## Context

Currently, `PRAGMA_SQL` is executed unconditionally on every adapter init. PostgreSQL does not support SQLite PRAGMAs, so this would fail with a syntax error if passed to the PostgreSQL adapter. The fix is a simple conditional: only execute PRAGMA_SQL when the adapter is NOT PostgreSQL.

## Implementation Steps

### Step 1: Locate the PRAGMA_SQL execution
```bash
grep -n "PRAGMA_SQL\|prgama" src/lib/db/index.js src/lib/db/driver.js
```

### Step 2: Add the guard
```js
// In the init function (src/lib/db/index.js or driver.js)
if (adapter.driver !== "postgres") {
  await adapter.exec(PRAGMA_SQL);
}
// PostgreSQL adapter's checkpoint() is a no-op, so no WAL timer is started
```

### Step 3: Verify no regression for SQLite paths
```bash
# Run app without POSTGRES_URL
npm start
# Verify in logs: PRAGMA_SQL executed (check for "WAL" or similar log output)
```

### Step 4: Verify no PRAGMA error for PostgreSQL
```bash
# Run app with POSTGRES_URL
POSTGRES_URL=postgres://test:test@localhost:5432/9router npm start
# Verify no SQL syntax error about PRAGMA
```

- This is a 3-line change but prevents a hard crash on startup with PostgreSQL
