## Context

apiKeysRepo has no JSON columns (apiKeys table has key, name, machineId, isActive, createdAt). The only change is removing any parseJson/stringifyJson and updating placeholder syntax. The critical concern is the `validateApiKey` auth hot path -- it must return `true`/`false` as Booleans, not `1`/`0` as numbers.
