## Task Description

Migrate `src/lib/db/repos/requestDetailsRepo.js` to use the PostgreSQL adapter. Remove parseJson from the `data` column. Verify the self-pruning DELETE subquery works in PostgreSQL and the UPSERT pattern `ON CONFLICT(id) DO UPDATE` works.

## Context

requestDetailsRepo stores full request/response payloads in a JSONB `data` column. The repo has:
- `saveRequestDetail()`: UPSERT by id
- `getRequestDetails()`: Paginated SELECT with ORDER BY timestamp DESC
- Self-pruning: DELETE old rows when COUNT(*) > maxRecords

The self-pruning uses a subquery that must work identically in PostgreSQL.
