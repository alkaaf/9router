## Task Description

Migrate the rules engine -- which stores rules in the `kv` table -- to use the PostgreSQL adapter. The rules engine reads/writes rules via the kvStore helper.

## Context

Rules (request routing rules, model mappings, etc.) are stored in kv with `scope = "rules"`. Since kvStore handles the SQL, this task is mostly verification of the kvStore migration.

## Implementation Steps

### Step 1: Find rules usage
```bash
grep -rn "rules\." src/lib/ --include="*.js" | grep -i "kv\|get\|set" | head -20
```

### Step 2: Verify rules CRUD
```js
// Rules should work identically with PG:
await kv.set("rules", "rule-1", { match: "gpt-4", target: "gpt-4-turbo" });
const rule = await kv.get("rules", "rule-1");
```

### Step 3: Run rules tests
```bash
POSTGRES_URL=... npm test -- --grep "rules"
```
