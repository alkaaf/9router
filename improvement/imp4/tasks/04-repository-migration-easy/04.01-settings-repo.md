## Task Description

Migrate `src/lib/db/repos/settingsRepo.js` to use the PostgreSQL adapter. Remove `parseJson`/`stringifyJson` calls since PostgreSQL JSONB handles serialization transparently. The `mergeWithDefaults()` logic stays unchanged.

## Context

settingsRepo.js currently uses `parseJson(row.data, {})` in `readRaw()` because SQLite returns JSON as TEXT that must be parsed. With PostgreSQL JSONB, `row.data` is already a JS object, so parsing is redundant. Similarly, `stringifyJson(merged)` in `updateSettings()` is unnecessary -- the adapter handles object-to-JSONB serialization.

## Implementation Steps

### Step 1: Read current settingsRepo.js
```bash
cat src/lib/db/repos/settingsRepo.js
# Identify parseJson and stringifyJson call sites
```

### Step 2: Remove parseJson from readRaw
```js
// BEFORE:
return mergeWithDefaults(parseJson(row.data, {}), defaults);

// AFTER:
return mergeWithDefaults(row.data, defaults);
// row.data is already an object from PostgreSQL JSONB
```

### Step 3: Remove stringifyJson from updateSettings
```js
// BEFORE:
const merged = { ...getCurrent(), ...updates };
await db.run("UPDATE settings SET data = ? WHERE id = 1", [stringifyJson(merged)]);

// AFTER:
const merged = { ...getCurrent(), ...updates };
await db.run("UPDATE settings SET data = $1 WHERE id = 1", [merged]);
// PostgreSQL adapter handles object-to-JSONB serialization
```

### Step 4: Update placeholder syntax
```js

```
