## Task Description

Migrate the rateLimits logic -- which is implemented via the `kvStore` helper -- to use the PostgreSQL adapter. Since rateLimits use the same kv composite-PK pattern, this task is mostly a verification of the kvStore migration.

## Context

The `kv` table has a composite primary key on (scope, key). Rate limits are stored with `scope = "rateLimits"` and key = some identifier. The kvStore helper handles the SQL -- this task verifies rate limits work with PG once kvStore is migrated.

## Implementation Steps

### Step 1: Find rateLimits usage
```bash
grep -rn "rateLimit\|rateLimit:" src/lib/ --include="*.js" | head -20
```

### Step 2: Verify kvStore usage
```js
// The rate limit code probably looks like:
const limits = await kv.get("rateLimits", key);
await kv.set("rateLimits", key, { requests: 100, period: 60000 });
// These should work identically with PG (kvStore handles the diff)
```

### Step 3: Test rate limit enforcement
- Set a low rate limit
- Exceed it
- Verify 429 response
- Verify kv is updated correctly in PG
