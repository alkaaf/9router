## Task Description

Verify that batch request tracking -- which writes to usageHistory and requestDetails via the queue -- works correctly with the PostgreSQL adapter. The usageRepo write path is rewired in Domain 5; this task is a sanity check on the integration with other repos.

## Context

Batch requests write usage records via the same flush pipeline as regular requests. The read paths (getBatchStatus, getBatchDetails) read from requestDetails. The write path is fully covered by Domain 5; this task verifies the read paths work.

## Implementation Steps

### Step 1: Find batch request handlers
```bash
grep -rn "batchRequest\|batch/" src/app/api/ --include="*.js" | head -20
```

### Step 2: Run batch integration tests
```bash
POSTGRES_URL=... npm test -- --grep "batch"
```

### Step 3: Manual smoke test
- Submit a batch request with 3 messages
- Verify batch status returns 'completed'
- Verify all 3 messages have usage records
- Verify requestDetails has the batch parent record
