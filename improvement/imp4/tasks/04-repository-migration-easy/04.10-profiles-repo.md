## Task Description

Migrate the kv-based repos (aliasRepo, pricingRepo, disabledModelsRepo) and the `kvStore`/`jsonCol` helpers to use the PostgreSQL adapter. The kv table has a composite primary key on (scope, key), and the upsert syntax `ON CONFLICT (scope, key) DO UPDATE` is identical in PostgreSQL.

## Context

The kv pattern is used by 4-5 features: alias (model aliases), pricing (model cost data), disabled models, rate limits, rules. All use the same kvStore helper. The migration is:
1. Update kvStore.js to skip parseJson/stringifyJson when using PG
2. Update jsonCol.js to add a guard: if PG adapter, return the value as-is (already an object)
3. Verify the 3 repos work with both backends

## Implementation Steps

### Step 1: Update jsonCol.js with PG guard
```js
// src/lib/db/helpers/jsonCol.js
export function parseJson(value, defaultValue) {
  if (value === null || value === undefined) return defaultValue;
  // PostgreSQL JSONB returns already-parsed objects
  if (typeof value === "object") return value;
  return JSON.parse(value);
}

export function stringifyJson(value) {
  if (value === null || value === undefined) return value;
  // PostgreSQL JSONB handles object serialization
  if (typeof value === "object") return value;
  return JSON.stringify(value);
}
```
