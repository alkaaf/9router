## Task Description

Migrate `src/lib/db/repos/connectionsRepo.js` to use the PostgreSQL adapter. Remove parseJson/stringifyJson from the `data` column handling. Verify `reorderInTx()` N+1 UPDATE loop works within an async transaction.

## Context

connectionsRepo stores provider connection configs with a JSONB `data` column. The data column contains connection-specific settings that are opaque to the repo -- the repo only reads/writes the whole blob. The `reorderInTx()` method reorders connections by priority using N+1 UPDATE statements inside a transaction. PostgreSQL handles this identically to SQLite for small row counts (<100).

## Implementation Steps

### Step 1: Read current connectionsRepo.js
```bash
cat src/lib/db/repos/connectionsRepo.js
```

### Step 2: Remove parseJson from data reads
```js
// BEFORE: return parseJson(row.data, {});
// AFTER: return row.data; // already an object from PG JSONB
```

### Step 3: Remove stringifyJson from data writes
```js
// BEFORE: await db.run("UPDATE ... SET data = ?", [stringifyJson(data)]);
// AFTER: await db.run("UPDATE ... SET data = $1", [data]);
```

### Step 4: Update placeholder syntax
```js
// Change all ? to $1, $2, ...
```

### Step 5: Verify reorderInTx
```js
// The N+1 UPDATE in a transaction should work identically
const result = await connectionsRepo.reorderInTx([id1, id2, id3]);
// Verify priorities are updated correctly
```
