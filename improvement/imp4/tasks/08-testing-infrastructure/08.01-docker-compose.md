## Task Description

Create `docker-compose.test.yml` with a `postgres:16-alpine` service for local integration testing. Uses tmpfs for fast ephemeral storage and a health check. The integration test suite connects via `POSTGRES_URL=postgres://test:test@localhost:5432/9router_test`.

## Context

Integration tests need a real PostgreSQL instance. The existing test infrastructure likely has SQLite tests but no PG tests. Docker Compose provides a reproducible PG instance that can be started before tests and torn down after.

## Implementation Steps

### Step 1: Create docker-compose.test.yml
```yaml
services:
  postgres:
    image: postgres:16-alpine
    container_name: 9router-test-pg
    environment:
      POSTGRES_DB: 9router_test
      POSTGRES_USER: test
      POSTGRES_PASSWORD: test
    ports:
      - "5432:5432"
    tmpfs:
      - /var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U test -d 9router_test"]
      interval: 2s
      timeout: 5s
      retries: 30
```

### Step 2: Add a test script to package.json
```json
"scripts": {
  "test:postgres:up": "docker compose -f docker-compose.test.yml up -d",
  "test:postgres:down": "docker compose -f docker-compose.test.yml down",
  "test:postgres": "npm run test:postgres:up && POSTGRES_URL=postgres://test:test@localhost:5432/9router_test npm test -- --grep postgres; npm run test:postgres:down"
}
```

### Step 3: Verify
```bash
npm run test:postgres:up
sleep 5
docker compose -f docker-compose.test.yml exec postgres psql -U test -d 9router_test -c "SELECT 1"
npm run test:postgres:down
```

## Success Criteria

- [ ] `docker compose -f docker-compose.test.yml up -d` starts PG
- [ ] Health check passes (`pg_isready`)
- [ ] Connection from host works on port 5432
- [ ] `npm run test:postgres:down` stops PG
- [ ] Test script can run a single test file

## Testing Checklist

- [ ] Start, connect, run query, stop
- [ ] Re-start works (no stale state from previous run)

## Notes

- tmpfs makes the PG instance ephemeral -- data is lost on stop
- For CI: use GitHub Actions' `services: postgres` (covered in 08.03)
- The `container_name` makes the container easy to find with `docker logs 9router-test-pg`
- Consider adding a `pgadmin` service for local debugging (optional)
