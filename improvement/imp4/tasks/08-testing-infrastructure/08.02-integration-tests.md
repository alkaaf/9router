## Task Description

Write integration tests for the PostgreSQL adapter and all migrated repositories. Tests run against the docker-compose.test.yml PostgreSQL instance. Use a `compareWithSqlite` helper to verify behavior matches the SQLite path.

## Context

Adapter and repository tests are the safety net for the migration. The pattern is: write the same test for both adapters, assert the outputs are identical. This catches any adapter-shape mismatches that would otherwise break the dashboard or API.

## Implementation Steps

### Step 1: Create test setup helper
```js
// tests/integration/postgres/setup.js
import { createPostgresAdapter } from "../../../src/lib/db/adapters/postgresAdapter.js";
import { TABLES_POSTGRES, INDEXES_POSTGRES, PRAGMA_SQL_POSTGRES } from "../../../src/lib/db/schema.postgres.js";

export async function setupPostgresTest() {
  const url = process.env.POSTGRES_URL;
  if (!url) throw new Error("POSTGRES_URL not set");
  const adapter = await createPostgresAdapter({ connectionString: url });
  await adapter.exec(PRAGMA_SQL_POSTGRES);
  for (const table of TABLES_POSTGRES) await adapter.exec(table);
  for (const index of INDEXES_POSTGRES) await adapter.exec(index);
  return adapter;
}

export async function teardownPostgresTest(adapter) {
  for (const table of TABLES_POSTGRES.reverse()) {
    await adapter.run(`TRUNCATE TABLE ${table} CASCADE`);
  }
  await adapter.close();
}
```

```js

- run: INSERT/UPDATE/DELETE return correct shapes

- checkpoint: no-op
- close: pool ends
- Placeholder translation: ? to $1, $2 for 0-10 params
- BOOLEAN: true/false
- JSONB: store/retrieve objects
- Concurrent: 10 parallel transactions on different keys
```
