## Task Description

Create `scripts/migrate-sqlite-to-postgres.js` — a one-time data migration script that copies all data from an existing SQLite database to a fresh PostgreSQL instance. Handles type conversion (TEXT timestamps -> TIMESTAMPTZ, INTEGER booleans -> BOOLEAN, TEXT JSON -> JSONB), streams large tables (usageHistory) to avoid loading everything into memory, and verifies row counts.

## Context

Operators upgrading from SQLite to PostgreSQL need a one-time data migration. The script:
1. Opens both databases
2. Creates the PG schema
3. Migrates small tables in bulk
4. Streams usageHistory in 10k batches
5. Populates the 5 normalized rollup tables from existing usageDaily JSON blobs
6. Verifies row counts

## Implementation Steps

### Step 1: Create script skeleton
```js
#!/usr/bin/env node
// scripts/migrate-sqlite-to-postgres.js
import Database from "better-sqlite3";
import { createPostgresAdapter } from "../src/lib/db/adapters/postgresAdapter.js";
import { TABLES_POSTGRES, INDEXES_POSTGRES, PRAGMA_SQL_POSTGRES } from "../src/lib/db/schema.postgres.js";

const args = parseArgs(process.argv.slice(2));
const sqlitePath = args.sqlite || "./data.db";
const pgUrl = args.postgres || process.env.POSTGRES_URL;
if (!pgUrl) { console.error("--postgres or POSTGRES_URL required"); process.exit(1); }

const sqlite = new Database(sqlitePath, { readonly: true });
const pg = await createPostgresAdapter({ connectionString: pgUrl });
```

### Step 2: Type conversion helpers
```js
function convertRow(row, tableName) {
  const out = { ...row };
  // Booleans: INTEGER 0/1 -> true/false
  for (const col of BOOLEAN_COLUMNS[tableName] || []) {
    if (out[col] !== null) out[col] = !!out[col];
  }
  // JSON: TEXT -> object
  for (const col of JSON_COLUMNS[tableName] || []) {
    if (out[col] && typeof out[col] === "string") out[col] = JSON.parse(out[col]);
  }
  // Timestamps: TEXT ISO strings pass through
  // Cost REAL -> pass through (NUMERIC(12,6) accepts number)
  return out;
}
```

### Step 4: Stream usageHistory
```js
const stmt = sqlite.prepare("SELECT * FROM usageHistory ORDER BY id");
let count = 0;
const BATCH = 10000;
let batch = [];
while (stmt.step()) {
  batch.push(convertRow(stmt.getAsObject(), "usageHistory"));
  if (batch.length >= BATCH) {
    await insertBatch(pg, "usageHistory", batch);
    count += batch.length;
    console.log(`  ${count} rows...`);
    batch = [];
  }
}
if (batch.length) await insertBatch(pg, "usageHistory", batch);
```

### Step 5: Migrate usageDaily and populate rollup tables
```js
// Load all usageDaily rows (typically <365)
const dailyRows = sqlite.prepare("SELECT * FROM usageDaily").all();
for (const row of dailyRows) {
  const data = JSON.parse(row.data);
  // Insert into usageDaily
  await pg.run("INSERT INTO usageDaily (dateKey, data) VALUES ($1, $2) ON CONFLICT (dateKey) DO UPDATE SET data = EXCLUDED.data", [row.dateKey, data]);
  // Populate rollup tables from JSON blob
  await populateRollupTables(pg, row.dateKey, data);
}
```

### Step 6: Verification
```js
// After migration:
for (const table of ALL_TABLES) {
  const sqliteCount = sqlite.prepare(`SELECT COUNT(*) as c FROM ${table}`).get().c;
  const pgCount = (await pg.get(`SELECT COUNT(*) as c FROM ${table}`)).c;
  if (sqliteCount !== pgCount) console.error(`MISMATCH: ${table}: sqlite=${sqliteCount}, pg=${pgCount}`);
  else console.log(`OK: ${table}: ${pgCount} rows`);
}
```

## Notes

- The script is one-time use; it does not need to be idempotent
- For large databases, consider running with `--limit` flag for incremental migration
- JSON blob parsing can fail on malformed data -- wrap in try/catch and log
- Document the migration in the runbook (Domain 9)
