## Description
Create the ProxyPool GORM model — proxy IP pool definitions with active/status filtering, JSON config data, and timestamps.

## Input
Existing Node schema: `id` (TEXT PK), `isActive` (BOOLEAN default true indexed), `testStatus` (TEXT nullable indexed), `data` (TEXT/JSONB), `createdAt` (timestamp), `updatedAt` (timestamp).

## Output
`internal/model/proxy_pool.go` with the `ProxyPool` struct.

```go
type ProxyPool struct {
    ID         string    `gorm:"primaryKey;type:text;column:id"`
    IsActive   *bool    `gorm:"default:true;index;column:isActive"`
    TestStatus *string  `gorm:"index;column:testStatus"`
    Data       string   `gorm:"not null;type:text;column:data"`
    CreatedAt  time.Time `gorm:"column:createdAt"`
    UpdatedAt  time.Time `gorm:"column:updatedAt"`
}
```

## Logic
1. `ID` is TEXT primary key — UUID generated by `BeforeCreate` hook (DB-011)
2. `IsActive` is `*bool` with `default:true` — `FindActive` (repo-level) filters `isActive = true`
3. `TestStatus` is a nullable indexed string — tracks health check results (e.g. "pass", "fail", "pending")
4. `Data` stores proxy configuration (host, port, auth credentials, rotation rules) as JSON
5. Register `BeforeCreate` and `BeforeUpdate` hooks for timestamp management (DB-011)
6. Two indexes: `isActive` (for `FindActive`) and `testStatus` (for `FindByStatus`)
